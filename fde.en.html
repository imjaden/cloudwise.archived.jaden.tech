📜 FDE History

Software's value doesn't live in the codebase — it lives in the customer's production environment.

Origin: Palantir's "Front-of-House Engineer" Experiment

FDE was pioneered by Palantir in the 2010s, inspired by high-end French restaurants — where front-of-house staff work closely with the kitchen to make sure every dish lands perfectly. The core insight: the hardest part of enterprise software isn't "building it" — it's "getting people to actually use it."

Key Milestones

2010-2015
Palantir noticed customers couldn't use, underused, or outright abandoned software after delivery. They started experimenting with the FDE model.
2016
Palantir's FDE headcount briefly exceeded their core software engineers — proving the model's commercial value.
2024 Q4
OpenAI starts building an FDE team. FDE goes from "Palantir thing" to "industry standard."
2025 Q1
Anthropic forms a $1.5B joint venture, hiring Applied AI Engineers (an FDE variant).
2026 Q1
OpenAI spins up "The Deployment Company" as a standalone entity, raising over $4B at roughly $10B valuation.
2026 Q2
FDE job postings hit 5,330 — an 800% increase. Deloitte, Alibaba Cloud, Databricks, Scale AI and others all have FDE roles.

🗺️ Learning Plan

Three-phase path: tech stack gap-fill (months 1-3) → business scenario practice (months 2-3) → interview prep (month 1)

Phase 1: Tech Stack Gap-Fill (Months 1-3)

Week 1-2: Python Full-Stack Level-Up

  • Week 1 Build RESTful APIs with FastAPI
  • Week 1 Data validation with Pydantic
  • Week 2 Database ops with SQLAlchemy + Alembic
  • Week 2 Async programming (asyncio)

Week 3-4: RAG Systems Hands-On

  • Week 3 Vector DB selection & deployment (pgvector / ChromaDB)
  • Week 3 Document chunking strategies (Recursive / Semantic)
  • Week 3 Embedding model selection (text-embedding-3-small / BGE)
  • Week 4 End-to-end RAG pipeline build
  • Week 4 Reranker integration

Week 5-8: Agent Development

  • Week 5 LangGraph basics (State Graph, Tool Node)
  • Week 6 MCP protocol & server development
  • Week 7 Multi-agent orchestration (CrewAI / AutoGen)
  • Week 8 Function Calling & Tool Use best practices

Week 9-12: Evaluation & Deployment

  • Week 9 Eval frameworks (RAGAS / DeepEval / custom)
  • Week 10 Docker + Docker Compose
  • Week 10 K8s basics (Deployment, Service, Ingress)
  • Week 11 vLLM / TGI inference serving
  • Week 12 Monitoring (Prometheus + Grafana) + Logging (ELK)

Phase 2: Business Scenario Practice (Months 2-3)

🏢 Scenario 1: Enterprise Knowledge Base RAG

Customer has PDF/Word docs, build a smart Q&A system.

Tech:Doc parsing → chunking → vectorize → retrieve → generate
Challenge:Multi-modal docs, access control, hallucination suppression
Deliverable:Runnable RAG service + evaluation report

💬 Scenario 2: Customer Service Agent

24/7 smart support agent that can look up orders, handle refunds, and answer product questions.

Tech:Intent recognition → tool calls → multi-turn dialogue → human fallback
Challenge:Edge cases, safety filtering, user experience
Deliverable:Agent service + admin dashboard

🔍 Scenario 3: Code Review Agent

AI-powered code review integrated into Git workflows.

Tech:Git integration → diff analysis → rule matching → comment generation
Challenge:Rule customization, false positive control, team collaboration
Deliverable:GitHub/GitLab bot + review reports

🛠️ Scenario 4: Ops Diagnostics Agent

Draw on AIOps experience to build a fault diagnosis agent.

Tech:Log analysis → metric correlation → root cause → fix suggestions
Challenge:Multi-source data fusion, real-time, accuracy
Deliverable:Diagnosis agent + visual dashboard

Case Study 5-Step Framework

  1. Clarify the problem and goals
  2. Identify stakeholders and success metrics
  3. Map out inputs and available data
  4. Break into solvable sub-problems, prioritize by risk/value
  5. Propose a "walking-skeleton" MVP, then iterate

Differentiation Storylines

  • 💬 "I have 14 years in tech, 6 as a pre-sales solution architect serving telco/UnionPay/aviation enterprise accounts"
  • 💬 "I'm transitioning from pre-sales to FDE because I realized the real value isn't in 'getting customers to sign' — it's in 'getting customers to use it'"
  • 💬 "I've already practiced AI deployment in ops scenarios (change conflict detection, fault diagnosis), and now I want to apply that skill to a broader set of problems"
  • 💬 "My edge: I can both understand the customer's business pain points AND write the code to deliver the solution"
